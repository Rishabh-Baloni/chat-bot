This page demonstrates runtime widget configuration. The widget is configured to use a dark theme and positioned in the bottom-left.

  position: 'bottom-left',

API Endpoints

  • GET /health - Health check with service status
  • POST /chat - Chat with debug info (if DEBUG_MODE=true)
  • POST /expand-knowledge - Protected with admin key

Test Knowledge Expansion (requires admin key):

curl -X POST http://localhost:8000/expand-knowledge \
  -H "Content-Type: application/json" \
  -H "X-Admin-Key: your_admin_key" \
  -d '{"raw_text": "Sample text", "source_tag": "test", "domain": "general"}'

Environment Variables for Production

  • DEBUG_MODE=true - Enable debug info in responses
  • EXPAND_KNOWLEDGE_ENABLED=false - Disable knowledge expansion in production
  • LOG_TOKEN_USAGE=true - Log token usage per request
  • ADMIN_API_KEY=secret - Protect knowledge expansion endpoint
